# Notification Decision

## Selected Architecture

`GitHub Pages → Cloudflare Worker → Formspree + Cloudflare Queue → личный Telegram-бот`.

Почему:

- GitHub Pages остаётся простым публичным фронтендом.
- В браузере и репозитории хранится только публичный адрес шлюза; токен бота и идентификатор чата находятся в Cloudflare Worker Secrets.
- Полный ответ можно восстановить из кабинета приёмника, даже если уведомление потерялось.
- Formspree остаётся основным хранилищем и почтовым каналом.
- Telegram получает копию через очередь: длинные ответы делятся на пронумерованные части, а временные ошибки повторяются автоматически.
- Шлюз проверяет origin, тип и размер запроса и применяет ограничение частоты по адресу отправителя.

## Verification

- Бот и целевой чат подтверждены контрольным `sendMessage`.
- Серверная контрольная заявка успешно сохранена в Formspree и принята Telegram.
- Очередь и повторные попытки развёрнуты на Cloudflare; секреты отсутствуют в GitHub.

## Operations

1. Formspree и адрес уведомлений подтверждены 2026-07-15.
2. Cloudflare Worker, очередь повторов и серверные секреты подключены 2026-07-15.
3. GitHub Pages получает только `NEXT_PUBLIC_SUBMISSION_ENDPOINT`.
4. Если публичная форма начнёт получать распределённый спам, добавить Cloudflare Turnstile; текущая защита уже включает строгую проверку запроса и ограничение частоты.

## Rejected Patterns

- Telegram Bot API token, GitHub PAT, почтовый пароль или другой секрет в браузере.
- Публичные GitHub issues как хранилище клиентского брифа.
- `mailto:` как единственный путь отправки без readback и надёжной обработки ошибок.
