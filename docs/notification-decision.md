# Notification Decision

## Selected Baseline

`GitHub Pages → Formspree → подтверждённая почта владельца`.

Почему:

- GitHub Pages остаётся простым публичным фронтендом.
- В репозитории хранится только публичный ID формы, а не email-пароль, Telegram-токен или GitHub-токен.
- Полный ответ можно восстановить из кабинета приёмника, даже если уведомление потерялось.
- Почтовое приложение даёт push-уведомление на телефон без дополнительного backend-развёртывания.

## Optional Upgrade

Telegram и Google Sheets подключаются владельцем внутри приёмника формы, если нужен отдельный push-канал и удобная таблица. Полный текст лучше не дублировать в Telegram без отдельного решения по конфиденциальности; безопаснее отправлять короткое уведомление и открывать полный ответ в кабинете.

## One-time Owner Action

1. Создать форму в Formspree и подтвердить адрес уведомлений.
2. Передать только публичный endpoint вида `https://formspree.io/f/...`.
3. Ограничить форму финальным GitHub Pages доменом и включить доступную антиспам-защиту.
4. После подключения выполнить тестовую отправку и проверить кабинет плюс уведомление.

## Rejected Patterns

- Telegram Bot API token, GitHub PAT, почтовый пароль или другой секрет в браузере.
- Публичные GitHub issues как хранилище клиентского брифа.
- `mailto:` как единственный путь отправки без readback и надёжной обработки ошибок.
