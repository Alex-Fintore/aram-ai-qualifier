# State

## Current Focus

Передать клиенту публичный короткий бриф и дождаться первой реальной заявки.

## Confirmed Facts

- Проект остаётся `incubating` до коммерческого решения заказчика.
- Исходный бриф содержит пять тематических блоков, таблицу воронки, критерии пилота и список материалов.
- Первичная веб-форма содержит десять вопросов на одной странице, из которых обязательны пять.
- После успешной первой отправки появляется необязательный углублённый бриф; до этого его поля не отрисовываются и не входят в первую отправку.
- Целевой публичный хостинг — GitHub Pages.
- Публичный адрес: `https://alex-fintore.github.io/ai-qualifier/`.
- Безопасный канал ответов: публичная форма вызывает Cloudflare Worker, оригинал сохраняется в Formspree и уходит на подтверждённую почту, а копия доставляется в личный Telegram-бот.
- Аккаунт приёмника и адрес уведомлений подтверждены; форма `AI-квалификатор — клиентский бриф` создана.
- Публичный адрес шлюза подключён через переменную репозитория `NEXT_PUBLIC_SUBMISSION_ENDPOINT`; токен бота и идентификатор чата хранятся только в Cloudflare Worker Secrets.
- Telegram-доставка использует Cloudflare Queue: временные ошибки повторяются, длинные ответы делятся на пронумерованные части без обрезания.
- Шлюз принимает только ожидаемый origin и схему данных, ограничивает размер заявки и частоту отправок по адресу источника.
- Короткий первичный бриф опубликован на основном GitHub Pages URL.
- Публичный репозиторий переименован в `Alex-Fintore/ai-qualifier`; старый Pages URL больше не используется.
