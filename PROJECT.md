# AI-квалификатор входящих лидов

## Purpose

Клиентский сайт-бриф для первого знакомства и первичной оценки пилота голосового AI-квалификатора. Базовые ответы отправляются сразу, а подробности по воронке, интеграциям и пилоту клиент может добавить после этого по желанию.

## Audience

- Руководитель продаж, операционного блока или владелец проекта — для короткого первичного брифа.
- IT/CRM, телефония, юрист или специалист по безопасности — только если клиент решит дополнить углублённый бриф.

## Delivery Shape

- Публичная адаптивная форма на GitHub Pages.
- Десять вопросов на одной странице, из них пять обязательных.
- Первая отправка сразу сохраняет основные данные; только после успеха появляется необязательный углублённый бриф.
- Короткий и углублённый черновики хранятся только в браузере заполняющего.
- Получатель ответов подключается через внешний приёмник без секретов в публичном коде.

## Safety Invariants

- Не собирать персональные данные клиентов, записи разговоров, пароли или ключи доступа через публичную форму.
- Большие и чувствительные материалы запрашивать отдельно через согласованный безопасный канал.
- Исходный DOCX и расшифровка не входят в публичный репозиторий.
