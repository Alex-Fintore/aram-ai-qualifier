# AI-квалификатор входящих лидов

Клиентский сайт-бриф для подготовки пилота голосового помощника: воронка, сценарий звонка, интеграции, требования к данным, критерии успеха и бюджет.

Публичная версия: `https://alex-fintore.github.io/ai-qualifier/`.

## Возможности

- десять коротких вопросов на одной странице, из них пять обязательных;
- необязательный углублённый бриф только после отправки основных данных;
- локальное сохранение черновика в браузере;
- адаптивный интерфейс и доступная клавиатурная навигация;
- защищённая отправка с сохранением в Formspree, почтовым уведомлением и копией в Telegram;
- серверная очередь с повторными попытками доставки Telegram и разбиением длинных ответов на части.

## Локальный запуск

```bash
npm install
npm run dev
npm test
npm run pages:build
```

## Канал ответов

Сборка читает публичный адрес шлюза из `NEXT_PUBLIC_SUBMISSION_ENDPOINT`. В GitHub Actions он передаётся из одноимённой переменной репозитория. Пока переменная не задана, кнопка сетевой отправки честно отключена.

Шлюз находится в `workers/telegram-relay/`. Он принимает запросы только с опубликованного GitHub Pages origin, ограничивает частоту и размер данных, сначала сохраняет исходную заявку в Formspree, а затем ставит Telegram-копию в очередь с повторными попытками. Токен бота и идентификатор чата хранятся только как Cloudflare Worker Secrets и не попадают в сборку сайта или GitHub.

Не добавляйте в репозиторий исходный DOCX, расшифровку, клиентские данные, записи разговоров, `.env` или любые токены.
